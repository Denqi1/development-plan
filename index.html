02
4-месячный роадмап
Месяц 1
React углублённо + react-query
Это база. Без уверенного React остальное строить сложнее. Параллельно — регулярные выражения (небольшая тема, 2–3 недели).
Месяц 2
Web API + кеш и сеть
Intersection Observer, Workers, IndexedDB, History. Браузерный и сетевой кеш, нестандартные запросы. Начать обновлять резюме и первые отклики.
Месяц 3
Node.js + Express/Fastify
Написать простой API. Понять как работает сервер изнутри — это сильно прокачает понимание фронтенд-задач. Активный поиск работы.
Месяц 4
Docker + финальное закрепление
Базовый Docker: образы, контейнеры, docker-compose. Повторение всего пройденного. Подготовка к разговору о повышении с конкретными примерами.
02.5
React · план на месяц
Стек: Vite + React + TypeScript + React Query + json-server  ·  Проект: mini-CRM (контакты + задачи) — с первого дня, параллельно теории
Неделя 1 Фундамент React
критично
useCallback, useMemo, React.memo
Referential equality, когда нужны, когда вредят. Профилировать через DevTools — найти реальный лишний ре-рендер и починить.
критично
useEffect — продвинутый уровень
Cleanup функции, отмена fetch через AbortController, race conditions. Понять когда useEffect — не лучший выбор.
важно
useReducer + useContext
Локальный state management без Redux. Когда useReducer лучше useState. Статусы задач в проекте — через useReducer.
Проект · Список контактов с поиском и фильтром — без оптимизации. В конце недели профилируешь и чинишь.
Неделя 2 Архитектура + TypeScript
критично
Паттерны: compound components, render props
Это главное что отличает мидла. Написать ContactCard как compound component (Root, Header, Tasks) — понять почему это лучше prop drilling.
критично
Проектирование кастомных хуков
Уровень выше: думать об API хука до написания кода. useSearch и useFilter — типизировать через дженерики.
важно
TypeScript: дженерики и utility types
Generic компоненты и хуки. Partial, Pick, Omit, ReturnType — не зубрить, а понять зачем. Практика: типизировать свои хуки.
Проект · Карточка контакта как compound component. Хуки useSearch / useFilter с дженерик-типизацией.
Неделя 3 React Query с нуля
критично
useQuery, useMutation, кэш
Базовые запросы, staleTime, gcTime, loading/error состояния. Заменить useEffect+fetch на useQuery — сразу почувствуешь разницу.
критично
Инвалидация и зависимые запросы
invalidateQueries после мутации, enabled флаг, запросы зависящие от других. Задачи загружаются только после загрузки контакта.
дополнительно
Оптимистичные обновления, prefetching
Если успеешь — сильно повысит уровень. Если нет — оставь на после месяца.
Проект · Заменить все useEffect+fetch на useQuery. useMutation для создания/редактирования задач с инвалидацией.
Неделя 4 Тесты + шлифовка
критично
React Testing Library
Тесты на свои хуки (useSearch, useFilter) и ключевые компоненты. Не покрытие ради покрытия — понять что тестировать и почему.
важно
Закрыть пробелы из недель 1–3
По ходу проекта сам увидишь что не усвоилось. Неделя 4 — буфер для этого. Не пытайся добавить новые темы.
Проект · Финальная версия mini-CRM. Принимать архитектурные решения самостоятельно — не по образцу.

05
Без выгорания
1
Одна тема за раз
Не учи Node.js и Web API одновременно. Доводи тему до уровня "могу объяснить" — потом следующая.
2
Практика важнее чтения
Прочитал — сразу напиши код. Даже 10 строк. Без практики знания не закрепляются.
3
Воскресенье — священно
Полный отдых от кода. Это не лень — это топливо для остальных шести дней.
4
Если пропустил — не догоняй
Пропустил день? Просто продолжай завтра. Попытка "навёрстывать" — путь к выгоранию за 2 недели.
5
Тема для митапа = тема для грейда
Выбери то, что изучаешь прямо сейчас. Подготовка к выступлению углубляет знания — не тратишь время дважды.
6
Английский — только как привычка
15 минут утром или в обед. Подкаст, статья, Duolingo. Не курс — просто ежедневный контакт с языком.
